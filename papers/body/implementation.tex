\chapter{系统实现与检测流程}
\label{chap:implementation}
\defaultfont

\section{系统模块划分}
基于 WiFi CSI 的脊柱姿态检测系统由数据读取模块、特征处理模块、文件级特征构建模块、模型训练模块和检测输出模块组成。各模块按照数据流顺序连接，形成从 CSI 原始数据到脊柱姿态类别输出的完整处理流程。

数据读取模块负责解析 CSI 原始文件，提取时间帧、子载波、接收天线和发射天线信息；特征处理模块负责计算接收天线相位比并进行时间相位展开；文件级特征构建模块通过滑动窗口和统计聚合生成 1620 维特征向量；模型训练模块使用带标签文件训练 ExtraTrees 分类器；检测输出模块根据训练完成的模型输出最终姿态类别。

\section{代码结构}
当前项目采用 Python 实现主要数据处理和模型训练流程。CSI 解析、特征构建、训练和评估分别由不同模块负责，便于单独调试和复现实验结果。主要代码入口包括：
\begin{itemize}
\item \texttt{src/wispine/csi.py}：负责 CSI 原始数据解析；
\item \texttt{src/wispine/dataset.py}：负责数据划分和窗口缓存；
\item \texttt{src/wispine/features.py}：负责相位比与文件级统计特征构建；
\item \texttt{scripts/train\_file\_classifier.py}：负责 ExtraTrees 模型训练；
\item \texttt{scripts/evaluate\_file\_classifier.py}：负责模型评估与指标输出。
\end{itemize}

\section{训练流程}
训练阶段首先读取带标签的 CSI 文件，并按照固定随机种子在文件级划分训练集、验证集和测试集。随后，系统对每个文件执行 CSI 解析、接收天线相位比计算、相位展开、滑动窗口生成和文件级统计特征聚合，得到用于分类的 1620 维特征向量。

在模型训练阶段，ExtraTrees 分类器使用训练集文件特征学习分类规则，并通过验证集比较不同配置的表现。最终模型使用 700 棵极端随机树、类别平衡权重、平方根特征采样和固定随机种子。训练完成后，系统保存模型文件和实验指标，便于后续复测和部署。

训练输出目录中主要包含以下文件：
\begin{itemize}
\item \texttt{model.joblib}：训练完成的 ExtraTrees 模型；
\item \texttt{metrics.json}：训练配置、数据划分和主要评价指标；
\item \texttt{evaluation\_test.json}：测试集评估结果；
\item \texttt{evaluation\_test\_local.json}：本地重载模型后的复测结果。
\end{itemize}

\section{检测流程}
检测阶段对新的 CSI 文件执行与训练阶段一致的数据处理流程。系统首先解析复数 CSI，然后以第 0 根接收天线为参考计算相位比，并沿时间维度展开相位；之后生成长度为 512、步长为 256 的滑动窗口，并将该文件的全部窗口聚合为 1620 维统计特征；最后加载 \texttt{model.joblib}，由 700 棵树组成的 ExtraTrees 模型输出三类姿态之一。

检测流程可表示为
\begin{equation}
X_{\mathrm{raw}}\rightarrow X_{\mathrm{csi}}\rightarrow X_{\mathrm{ratio}}\rightarrow X_{\mathrm{stat}}\rightarrow \hat{y}.
\end{equation}
其中 $X_{\mathrm{ratio}}$ 表示相位比序列，$X_{\mathrm{stat}}$ 表示文件级统计特征，$\hat{y}$ 表示最终检测类别。

\section{复测结果}
模型训练完成后，对保存模型进行了本地复测，以验证模型文件、数据处理流程和评估脚本的一致性。

复测结果与训练报告中的云端测试结果一致，均为 92.04\% 的测试准确率和 92.08\% 的宏平均 F1 值，混淆矩阵也保持一致。这说明保存的模型文件、数据处理流程和本地评估脚本能够复现实验结果。

\section{应用特点}
该系统不依赖摄像头图像和可穿戴设备，能够在室内 WiFi 环境下进行非接触式姿态感知。系统通过 CSI 相位比特征捕获人体姿态对无线信道的影响，并利用文件级统计特征和 ExtraTrees 集成分类器完成三类姿态检测，具有隐私友好、部署成本低和计算开销较小等特点。
